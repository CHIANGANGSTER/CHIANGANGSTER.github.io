四模型图像工作流

单图使用 / 多图联结

这组工作流包含 Seedream、Nano Banana Pro 2、Flux.2 Pro、GPT Image 2 四个版本。四个版本的使用方式一致，只是底层模型与节点配置不同。 视频演示使用 GPT Image 2 工作流录制；其他三个工作流按照同样步骤操作即可。单图模式适合一张参考图，多图联结适合把主体、场景、风格或结构拆成多张图输入。

单图生成

只有一张产品图、人物图、场景图或参考图时使用。把图片接入对应输入节点，保持默认连接关系，先跑通基础生成。

多图联结

把主体、参考风格、场景氛围或局部细节分配到多张输入图，让模型在同一次生成中融合多图信息。

工作流定位

One usage pattern across four image models

这页不是单独讲某一个模型，而是统一说明四个图像工作流的使用方式。Seedream、Nano Banana Pro 2、Flux.2 Pro、GPT Image 2 都按同一套输入逻辑操作：先判断是单图还是多图，再根据目标把参考图放入对应输入位置。

两种输入方式

Single image first, multi-image when relationship matters

单图

适合只有一个明确参考对象时使用。输入图承担主体、构图、材质或风格参考，工作流负责生成新的画面结果。

多图联结

适合把多个信息源合并：一张给主体，一张给场景，一张给风格，一张给细节。每张图都要有清晰职责。

共用逻辑

四个工作流的入口和操作顺序保持一致。区别只在模型能力、节点配置和输出倾向。

推荐顺序

先用单图跑通，再切到多图联结。这样更容易判断问题来自模型、输入图还是多图关系。

操作要点

Keep inputs clean and relationships explicit

准备输入图

单图模式只需要一张清晰参考图；多图模式需要提前决定每张图的角色，避免多张图同时表达相互冲突的主体、构图或风格。

先跑默认连接

不要一开始就大幅调整节点。先按默认连接跑一版，确认图像能正常进入工作流，再根据结果调整输入图和提示词。
