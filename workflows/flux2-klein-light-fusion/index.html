FLUX.2 KLEIN 光影融合

溶图打光工作流

这个工作流面向产品图、参考图和场景草图的光影融合：尽量保留原图的构图、主体比例和细节，同时重打光线、阴影、反射与局部透视遮挡。 它适合把白底产品、现有素材或设计草图快速融入更完整的商业场景，用较短链路确认光感、氛围和可继续精修的方向。 核心不是一次出最终成片，而是先把“主体不乱、光影统一、场景可信”跑稳定，再进入局部修图、放大或产品精修流程。

工作流定位

Relight and blend images without rebuilding everything from zero

这条链路更像“产品溶图和重打光”的中间工序。输入图负责提供主体结构、原始透视和基础细节；工作流里的参考潜空间、遮罩、LoRA 和采样参数负责统一光源方向、阴影状态、反射规律和局部遮挡。它特别适合家具、家居、产品海报、电商场景和需要统一光感的合成图。

适用任务

把已有产品或参考图放进新的光影环境里，修正阴影、遮挡、材质反射和局部透视，让主体与场景看起来像同一张照片。

不适合任务

如果需要从零生成完整场景，优先使用 Klein 文生图页面；如果已经选定最终画面并需要商业级收尾，再进入产品精修工作流。

核心节点与参数

What the attached workflow actually uses

模型链路

UNET 使用 FLUX.2 Klein 9B FP8，CLIP 使用 qwen_3_8b_fp8mixed，VAE 使用 flux2-vae。整体配置偏轻，方便快速验证溶图与打光方向。

重打光 LoRA

工作流加载“溶图重打光自纠正透视”LoRA，强度为 0.85。它负责强化光影融合、透视校正和局部遮挡关系，是这条流程的关键控制点。

参考潜空间

输入图先经过比例缩放、VAEEncode 和 ReferenceLatent，用来约束主体位置、构图和基础细节，减少模型把原图结构重新画乱的概率。

抠图与边缘

BiRefNet-General + VITMatte 负责提取主体遮罩，MaskGrow 当前为 -3，用轻微收缩处理边缘，避免溶图时出现发白、脏边或浮在场景上的感觉。

采样设置

KSampler 当前为 steps 4、CFG 1.2、Euler、Simple、Denoise 1。它强调速度和方向验证，适合先多跑几版，再挑图进入精修。

细节恢复

HLFrequencyDetailRestore 用于补回高频细节，让产品边缘、材质纹理和局部层次更稳，减少重打光后画面发糊的问题。

输入图与遮罩准备

Let the workflow read structure, matte and light direction

这个流程的重点不是让同事额外写文字，而是把输入图准备干净，让 ReferenceLatent 读到稳定构图，让 BiRefNet 抠出可靠主体，让重打光 LoRA 只处理光影、遮挡、投影和反射。前期把图片条件处理好，比临时补充描述更稳定。

主体图

选择轮廓完整、遮挡少、主体边缘清楚的图片。产品不要贴边，也不要被复杂背景吞掉，否则遮罩和透视约束都会变得不稳定。

参考光感

用参考图或当前场景确定主光方向、软硬程度、阴影落点和反射强弱。目标不是重画主体，而是让主体像被同一个现场光源照亮。

遮罩边缘

当前 MaskGrow 为 -3，适合轻微收边，减少白边和脏边。如果主体边缘被吃掉，就调回更温和的数值，再重新跑一版。

尺寸比例

先保持输入图的合理比例，再用 ImageScaleByAspectRatio 控制长边尺寸。不要一开始就压到过小，否则材质纹理和边缘层次会丢失。

推荐使用步骤

A practical relighting loop

准备输入图

先选一张主体清晰、轮廓完整、透视关系可用的产品图或参考图。输入图越干净，BiRefNet 遮罩和 ReferenceLatent 的约束越稳定。

确定光影目标

先定参考光感：主光从哪来、阴影落在哪里、反射应该强还是弱、背景需要保留多少。这个步骤由参考图和节点参数承担，同事只需要替换输入图并确认目标光感。

小批量跑图

保持输入图、尺寸和 LoRA 强度不变，先改 seed 或少量节点参数，一次跑多张。第一轮只看主体是否稳、阴影是否贴地、光感是否统一。

挑图后精修

选到方向后，再把结果交给产品精修、局部重绘或放大链路。Klein 负责快速找到融合方向，精修流程负责把画面推到交付标准。

优势与注意事项

Why this workflow is useful for design production

出图快

4 steps 配合 Klein 9B，适合前期快速试光、试背景、试风格，不必每次都走高成本精修链路。

主体更稳

ReferenceLatent 与遮罩链路共同约束主体，能降低产品轮廓被重绘、比例漂移或细节丢失的概率。

真实性更强

ReferenceLatent、遮罩和重打光 LoRA 都围绕光影、投影、反射和遮挡展开，输出重点是“像合成进同一环境”，不是单纯换风格。

仍需筛选

这不是最终修图按钮。遇到边缘脏、阴影错位或材质过度变化时，先回到输入图、遮罩、LoRA 强度和 seed，而不是直接放大。

文件与交付

Keep the workflow reusable

工作流 JSON

页面顶部按钮可下载附件中的 ComfyUI JSON。导入后检查本机模型名、LoRA 名称、VAE 路径和自定义节点是否一致。

建议保存内容

每次稳定输出后，保存原图、参考图、seed、输出图和对应 JSON 版本。这样下次只需要替换输入图或调整节点参数，就能复用同一套打光逻辑。
