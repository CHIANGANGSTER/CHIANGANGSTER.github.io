FLUX.2 KLEIN 9B

文生图场景搭建工作流

这是一个面向产品场景搭建的 ComfyUI 文生图工作流。 它可以结合 prompts 页面里的结构化提示词库，或使用浏览器插件反推参考图提示词，快速生成室内、电商、空间氛围和产品展示场景。 核心目标不是炫技，而是让设计师用更短链路得到可继续筛选、放大和交付的场景初稿。

工作流定位

From prompt structure to scene generation

这个流程适合从零生成产品场景：先用提示词库或插件把参考图拆成空间、主体、材质、光影、构图等结构，再把这些信息交给 Flux.2 Klein 9B。它更像一个快速场景草图机，适合做方向探索、客户提案、产品氛围图和批量风格测试。

提示词来源

可以直接使用 prompts 页面里的 JSON PROMPT，也可以用 Framia Clipper 在浏览器里反推任意参考图的提示词结构，再改成自己的产品场景。

输出目标

生成有空间感、材质关系和商业摄影质感的场景图，用于筛选构图、测试风格、建立后续精修或放大的基础。

核心节点与参数

Klein 9B quick generation path

模型

UNET 使用 FLUX.2 Klein 9B FP8 版本，CLIP 使用 qwen_3_8b_fp8mixed，VAE 使用 flux2-vae。整体链路偏轻，适合快速迭代。

采样

KSampler 当前配置为 steps 4、CFG 1.2、Euler、Simple。它强调速度和方向验证，适合先跑多张图挑方向。

LoRA 质感

工作流包含 UnifiedReward-Flex LoRA，作为质感和可用性增强。它帮助输出更接近商业图，而不是停留在粗糙草图阶段。

为什么适合场景搭建

Fast, open, realistic enough for design iteration

出图快

少步数采样让它更适合前期找方向。先用低成本判断空间、风格和产品气质，再决定是否进入精修链路。

开源链路

模型、VAE、LoRA 和节点都可以在本地 ComfyUI 管理，团队可以保存版本、复用工作流，并沉淀成内部资产。

真实性

Flux.2 Klein 对空间结构、光影层次和材质关系有较好的理解，适合做室内、电商、家具和生活方式场景。

提示词兼容

结构化提示词可以直接沿用 prompts 页的主体、空间、CMF、摄影和构图维度，减少每次从零写提示词的成本。

推荐使用步骤

A practical scene-building loop

整理参考

从 Pinterest、品牌页面、竞品图或 prompts 页面挑一个方向。先确认场景类型、主体位置、材质和光线，不要一开始就追求完整大段自然语言。

生成结构化提示词

用插件反推参考图，或复制 prompts 页面里的 JSON PROMPT。保留空间、CMF、摄影、构图、负面词这些关键字段，再替换成自己的产品和场景需求。

小批量跑图

固定宽高和核心提示词，先改 seed 和局部描述，一次跑多张。重点看构图、真实感和风格方向，而不是第一轮就看细节完美度。

进入精修

选出方向后，再把结果交给产品精修、局部重绘、放大或后期修图流程。Klein 负责找方向，精修流程负责把图推到交付标准。

文件与交付

Keep the workflow reusable

工作流文件

页面顶部的下载按钮会保存当前 Flux.2 Klein 文生图 JSON。导入 ComfyUI 后，按本机模型路径调整模型名、LoRA 和 VAE。

提示词资产

建议把稳定输出的提示词保存回 prompts 页面对应分类，记录场景、CMF、构图和负面词，方便下次直接复用。
